# H2 2025 Recommendation

### Restore the 20/65/15 balance before velocity compounds downward.

<div class="grid grid-cols-3 gap-6 mt-4">

<div>

#### 1. Rebalance Allocation
**Sprints 07–12**
- Move to **20% tech debt / 65% feature / 15% other**
- Reserve first 2 days of each sprint for debt items
- Eliminate top-5 oldest architecture debt items by Sprint 10
- Freeze new debt: every deprecated API addition requires offsetting retirement plan

</div>

<div>

#### 2. Stabilize Sprint Health
**Sprints 07–09**
- Enforce 10% sprint buffer for estimation variance
- Require cross-team dependency resolution **before** sprint commitment
- Cap mid-sprint scope additions at **5%** of planned points
- Track and report carry-over root cause per sprint

</div>

<div>

#### 3. Quality Reset
**Sprints 07–10**
- **Sprint 07:** Flaky test quarantine + CI hardening (est. 15 pts)
- **Sprints 08–09:** Close coverage gaps on payment & auth modules (est. 22 pts)
- Mandate PR review SLA: ≤8h, enforce via automated reassignment
- Reduce avg PR size: require split at 300 lines

</div>

</div>
