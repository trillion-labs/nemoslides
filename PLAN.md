# 48-Hour Execution Plan

Solo, 2 days, plenty of GPU. Demo-gallery focused (not live generation). Track B.

## Success criteria

1. **Baseline measured first.** Stock `Nemotron-3-Nano-30B-A3B-Base-BF16` scored on the 50-prompt held-out set with the PPTEval rubric (Content / Design / Coherence, 1–5 each) via VLM judge — this is the "status quo" number the project must beat.
2. **Finetuned model beats the baseline** on the *exact same* 50 prompts, judge, and rubric. Soft targets: aggregate Δ ≥ +0.5 on Design, ≥ +0.3 on Content and Coherence; pairwise win-rate > 60%.
3. Working demo gallery: ~10 side-by-side (base vs finetuned) decks, plus a short comparison reel.
4. Reproducible codebase: one-command data synthesis, one-command baseline eval, one-command training, one-command finetuned eval.
5. Pitch deck for the judges — **generated by the finetuned model itself**, as peer-vote bait.

## Sequencing rules

- **Do not launch SFT until hour 8.** Train only after the full downstream pipeline (synthesis → filter → render → judge → gallery) has been validated end-to-end using just the base model. If SFT explodes, the project still presents.
- **Baseline is a Track-B gate.** We cannot claim "the finetuned model is better" without concrete base-model numbers on the exact same held-out prompts under the exact same judge. The base-model PPTEval run at hour 7 is the status-quo measurement — this number is the reference point for the entire project's success claim. If it's missing, Track B's "validity of metrics" collapses.
- **Quick base-model competence check at hour 1** (before synthesis). Hit stock Nemotron-3-Nano-30B-A3B with 2–3 slide-generation prompts. This tells us how far the model has to travel (is it emitting garbage? passable markdown? non-Slidev HTML?) and informs prompt engineering for the full baseline run.
- **Skip NeMo-RL `uv sync` at hour 0.** Defer install to hour 7 so it overlaps the base-model eval run — free wall-clock.
- **Teacher model is validated before committing to 3,000 generations.** A 20-minute sanity check at hour 1 gates the rest of the data budget.

## Risk log

| Risk | Likelihood | Mitigation |
|---|---|---|
| NeMo-RL install fails (flash-attn, submodules) | Medium | Hour 7 install; if broken by hour 9, fall back to NeMo-AutoModel SFT path (also NeMo component, rubric-compliant) |
| Slidev render throughput too slow for 3,000 decks | Medium | Parallel Playwright pool; fall back to Marp for synthesis format (same SFT target shape, faster render) |
| Teacher-model output quality floor too low | Low–Medium | Hour-1 sanity check gates this; upgrade teacher model if failure |
| Finetuned model doesn't clearly beat base | Medium | Pre-filter synthesis aggressively; more epochs on smaller high-quality subset; cherry-pick gallery hero cuts honestly |
| Training OOMs or diverges | Low | LoRA (not full FT), gradient checkpointing, start from published Nano-3 30B-A3B config |
| vLLM logprob bug with Nemotron hybrid | Low | Use vLLM ≥ 0.17.0; `seq_logprob_error_threshold: 2` fallback documented in Nano-3 guide |

## Scope cuts (if hour 32 looks tight)

1. Drop the self-generated pitch deck — write pitch manually instead.
2. Shrink held-out eval from 50 → 30 prompts.
3. Drop the comparison reel; keep the static gallery page.
4. Cut domain diversity to just pitch decks (drop tech talks + product launches).

**Do not cut:** PPTEval rubric scoring (Track B metric validity), the side-by-side gallery, reproducibility.
